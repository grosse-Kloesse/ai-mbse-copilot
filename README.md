# MBSE Copilot

A lightweight prototype for structure-aware MBSE retrieval and question answering on real model data.

## What it does

The project focuses on answering MBSE-specific questions such as:

- which package an element belongs to
- which diagram contains an element
- which higher-level system a block belongs to
- what elements a package contains
- what properties or relations an element has

The current direction is:

**question understanding -> retrieval planning -> evidence retrieval -> answer generation**

## Repository structure

```text
api/        FastAPI service
data/       raw and processed data
docs/       notes and documentation
eval/       evaluation scripts and query sets
ingest/     parsing, indexing, and data preparation
tools/      local testing scripts for retrieval / QA
outputs/    generated batch test reports
```

## Main scripts

- `tools/eal_embedding.py`  
  Main script for single-query structured retrieval and QA on real model data.

- `tools/final_answers.py`  
  Runs a predefined query set and prints only the final answers.

- `tools/batch_test_queries.py`  
  Runs batch tests and exports full reports, summaries, and JSON results.

## Example usage

Run a single structured query:

```bash
python tools/eal_embedding.py --query "In welchem Diagramm erscheint der E-Motor" --show_plan
```

Run final-answer checks:

```bash
python tools/final_answers.py
```

Run batch testing:

```bash
python tools/batch_test_queries.py
```

## Batch outputs

Generated by `tools/batch_test_queries.py`:

- `outputs/full_batch_report.txt`
- `outputs/batch_summary.txt`
- `outputs/batch_results.json`

## Current focus

The implementation currently combines:

- query intent detection
- target entity extraction
- slot detection for property questions
- retrieval planning
- structure-aware lookup
- embedding-based semantic fallback
- answer generation from retrieved evidence

## Goal

The goal is to move from pure embedding retrieval toward a more reliable structure-aware MBSE QA pipeline for real model artifacts.
